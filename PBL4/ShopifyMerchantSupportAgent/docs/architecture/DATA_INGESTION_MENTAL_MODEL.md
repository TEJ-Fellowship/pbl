- Opens each Shopify documentation URL
- Waits for JavaScript to render content
- Extracts only relevant content (removes navigation, ads, footers)
- Categorizes content by merchant skill level

## ğŸ¯ Key Concepts & Mental Models

### 1. **Semantic Chunking**

**Concept**: Split documents at natural boundaries that preserve meaning.

**Why?**

- Context preservation: Related information stays together
- Search quality: Better retrieval when chunks contain complete thoughts
- Code preservation: Code blocks remain intact (critical for API docs)

**How?**

- Prioritize headings (strong semantic breaks)
- Respect paragraph boundaries
- Use sentence boundaries only for large sections
- Add overlap to maintain context across chunks

### 2. **Merchant Level Classification**

**Concept**: Categorize content by technical difficulty.

**Levels:**

- **Beginner**: Getting started, basic setup, simple tutorials
- **Intermediate**: Operational tasks, standard features
- **Advanced**: APIs, GraphQL, custom development, integrations

**Impact:**

- Future: Filter results by user skill level
- Future: Adapt response complexity
- Current: Metadata for analytics

### 3. **Overlap Strategy**

**Concept**: Overlap chunks to maintain context.

**Why?**

- Questions might span chunk boundaries
- Context from previous chunk helps understanding
- Reduces information loss at boundaries

**How?**

- Last 200 tokens from previous chunk
- Prepend to current chunk
- Maintains semantic continuity

### 4. **Vector Embeddings**

**Concept**: Convert text to numerical vectors that capture meaning.

**Properties:**

- Similar texts â†’ similar vectors
- Cosine similarity measures semantic similarity
- 384 dimensions balance quality and speed

**Why Local Model?**

- No API costs or rate limits
- Faster processing
- Privacy (data stays local)

### 5. **Dual Storage**

**Concept**: Store in both file system and Pinecone.

**File System:**

- Human-readable JSON
- Easy debugging
- Backup/version control
- No embeddings (saves space)

**Pinecone:**

- Vector database
- Fast similarity search
- Metadata filtering
- Cloud-based (scalable)
